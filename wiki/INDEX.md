# LLM Wiki — 知识索引

## 这是什么

本 Wiki 是项目知识的蒸馏层。每次会话后 Agent 自动将新发现写入此处，下次会话优先从这里读取。

知识优先级：Wiki → claude-mem → memory/日志

## 知识结构

| 页面 | 内容 | 更新方式 |
|------|------|----------|
| `SCHEMA.md` | Wiki 规约（页面格式、链接规则） | 仅首次初始化 |
| `LOG.md` | 所有 wiki 操作的操作日志 | 每次修改时追加 |
| `ARCHITECTURE.md` | 系统架构、组件关系、数据流 | 架构变更时 |
| `DECISIONS.md` | 设计决策记录（上下文、选项、理由） | 新决策时 |
| `CONCEPTS.md` | 领域概念解释（非显而易见的） | 新概念出现时 |
| `PATTERNS.md` | 重复出现的模式、最佳实践、常见陷阱 | 模式识别时 |
| `sessions/` | 每次会话的知识摘要（自动生成） | 会话结束时 |

## 语义查找表

### 我该看哪里？

| 我想知道... | 看这里 |
|---|---|
| 项目是干什么的？ | `CONCEPTS.md` → 项目概述 |
| 架构是怎么设计的？ | `ARCHITECTURE.md` |
| 为什么做了某个决定？ | `DECISIONS.md` |
| 这个模式怎么用？ | `PATTERNS.md` |
| 上次会话发生了什么？ | `sessions/YYYY-MM-DD.md`（最近的） |
| 某个概念的定义？ | `CONCEPTS.md` |
| 我是不是见过类似问题？ | 搜索所有 wiki 页面 |

### 快速导航

- 所有 wiki 页面应包含 `[[wikilinks]]` 交叉引用
- 通过 `INDEX.md` 找到相关页面 → 读取 → 遵循 wiki 链接 → 合成回答
- 如果 wiki 不够，回退到 claude-mem 搜索
