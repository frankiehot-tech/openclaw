# MAREF沙箱环境设计文档

## 概述

MAREF（Meta-Adaptive Reflective Evolutionary Framework）沙箱环境是一个基于控制论原理的智能工作流演化系统。它集成了64卦状态系统作为其核心状态表示，应用超稳定性原则确保系统在演化过程中保持可控和可预测。

## 设计目标

1. **超稳定性**: 系统在扰动下保持稳定，状态转换满足格雷编码约束
2. **适应性**: 根据质量反馈动态调整演化策略
3. **反射性**: 系统能够自我监控和调整
4. **可观测性**: 完整的状态转换日志和质量指标跟踪

## 核心架构

### 1. 沙箱管理器 (SandboxManager)

**职责**:
- 管理64卦状态系统的生命周期
- 强制执行格雷编码约束
- 协调各组件间的交互

**关键接口**:
```python
class SandboxManager:
    def __init__(self, state_manager: IntegratedHexagramStateManager):
        self.state_manager = state_manager
        self.feedback_controller = FeedbackController(self)
        self.evolution_engine = EvolutionEngine(self)
        self.monitor = SandboxMonitor(self)
    
    def evolve(self, target_quality: float, max_iterations: int = 100) -> EvolutionResult:
        """驱动系统向目标质量演化"""
    
    def apply_constraint(self, new_state: str) -> bool:
        """应用超稳定性约束"""
    
    def get_system_state(self) -> SystemState:
        """获取完整的系统状态"""
```

### 2. 反馈控制器 (FeedbackController)

**职责**:
- 监控质量指标（6个维度）
- 计算控制信号
- 调整演化参数

**控制论原理**:
```
质量误差 = 目标质量 - 当前质量
控制信号 = Kp × 质量误差 + Ki × 累积误差 + Kd × 误差变化率
演化步长 = f(控制信号)  # 自适应调整
```

### 3. 演化引擎 (EvolutionEngine)

**职责**:
- 生成候选状态转换
- 评估转换成本-收益
- 选择最优演化路径

**演化策略**:
1. **贪心策略**: 选择立即质量提升最大的转换
2. **模拟退火**: 允许暂时质量下降以跳出局部最优
3. **遗传算法**: 组合多个质量维度

### 4. 监控系统 (SandboxMonitor)

**职责**:
- 记录所有状态转换
- 计算性能指标
- 生成可视化报告

**监控指标**:
- 转换成功率
- 平均质量提升/步
- 约束违反次数
- 系统稳定性指数

## 超稳定性实现

### 约束定义

1. **格雷编码约束**: 汉明距离必须为1
2. **质量边界约束**: 质量评分必须在[0, 10]范围内
3. **转换速率约束**: 单位时间内最大转换次数
4. **回滚机制**: 检测到不稳定时自动回滚到稳定状态

### 稳定性证明

```
定义: 系统状态向量 S = [q1, q2, ..., q6] ∈ {0,1}^6
定义: 质量函数 Q(S) = Σ(激活维度)/6 × 10
定义: 稳定性条件 ∀ΔS, |ΔQ| < ε

定理: 在格雷编码约束下，系统演化是渐进稳定的
证明: 每次转换只改变一个维度，质量变化最多为 10/6 ≈ 1.67
      ∴ |ΔQ| ≤ 1.67 < ε (取ε=2.0)
```

```python
def evolve_to_target(self, target_hetu: HetuState) -> List[str]:
    """演化到目标河图状态"""
    current_state = self.state_manager.current_state
    target_states = self.state_manager.get_states_by_hetu(target_hetu)
    
    # 使用缓存选择最近卦象
    nearest = self.state_manager.cache_manager.select_nearest_hexagram(
        current_state, target_hetu
    )
    
    # 使用缓存查找路径
    path = self.state_manager.cache_manager.find_path(current_state, nearest)
    
    return path
```

## 数据流设计

```
用户请求 → 沙箱管理器 → 反馈控制器
                              ↓
                      演化引擎 → 状态管理器
                              ↓
                      监控系统 → 可视化报告
```

### 请求处理流程

1. **解析请求**: 提取目标质量要求
2. **初始化演化**: 设置初始状态和约束
3. **迭代演化**: 
   - 反馈控制器计算控制信号
   - 演化引擎生成候选转换
   - 沙箱管理器验证约束
   - 执行转换并更新状态
4. **结果返回**: 返回演化路径和质量报告

## 容错设计

### 故障检测

1. **约束违反**: 实时监控格雷编码约束
2. **质量下降**: 检测意外的质量下降
3. **循环检测**: 防止状态循环

### 恢复策略

1. **自动回滚**: 检测到故障时回滚到上一个稳定状态
2. **降级策略**: 切换到更保守的演化策略
3. **人工干预**: 严重故障时暂停并等待人工干预
