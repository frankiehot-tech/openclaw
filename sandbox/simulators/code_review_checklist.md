# 64卦状态系统代码审查清单

## 概述
本清单用于64卦状态系统及相关代码的审查，确保代码质量、一致性和可维护性。

## 基本审查项

### 1. 代码质量
- [ ] **代码格式化**: 使用black格式化，行长度≤88字符
- [ ] **导入排序**: 使用isort按black配置排序
- [ ] **代码检查**: 通过flake8检查，无E、F、W错误
- [ ] **类型提示**: 使用mypy进行类型检查，关键函数有类型注解
- [ ] **复杂度控制**: 函数圈复杂度≤10（flake8 max-complexity设置）

### 2. 测试要求
- [ ] **测试覆盖率**: 新代码测试覆盖率≥40%
- [ ] **单元测试**: 每个关键函数有对应的单元测试
- [ ] **集成测试**: 系统集成测试覆盖主要工作流
- [ ] **边界条件**: 测试覆盖边界情况和异常场景
- [ ] **测试命名**: 测试函数名清晰描述测试意图

### 3. 64卦状态系统特定要求
- [ ] **状态转换**: 状态转移符合格雷编码约束（汉明距离=1）
- [ ] **卦象映射**: 河图10态到64卦映射正确（参考hetu_hexagram_mapping.json）
- [ ] **质量维度**: 6个质量维度（correctness, complexity, style, readability, maintainability, cost_efficiency）正确计算
- [ ] **状态分析**: StateAnalysis对象包含完整的卦象分析信息
- [ ] **错误处理**: 状态转换失败有适当的错误处理和日志

### 4. 文档要求
- [ ] **函数文档**: 复杂函数有docstring说明
- [ ] **类型注解**: 函数参数和返回值有类型提示
- [ ] **注释**: 复杂逻辑有适当注释说明"为什么"
- [ ] **API文档**: 公共API有清晰的文档说明
- [ ] **更新日志**: 重大更改更新CHANGELOG.md

### 5. 安全与性能
- [ ] **安全检查**: 通过bandit安全检查，无高危漏洞
- [ ] **输入验证**: 用户输入和外部数据有验证
- [ ] **错误信息**: 错误信息不泄露敏感信息
- [ ] **性能考虑**: 关键路径有性能考虑，无明显的性能问题
- [ ] **内存使用**: 大对象有适当的内存管理

## 审查流程

### 提交前检查
1. 运行预提交钩子：`pre-commit run --all-files`
2. 运行测试套件：`pytest test_*.py`
3. 验证覆盖率：`pytest --cov=. --cov-report=term test_*.py`
4. 手动运行增强调度器测试：`python -m pytest test_enhanced_scheduler_integration.py -v`

### 审查会议前准备
1. 确保所有自动检查通过
2. 准备代码变更摘要
3. 识别潜在风险点
4. 准备测试结果和覆盖率报告

### 审查要点
1. **架构一致性**: 变更是否符合系统架构设计
2. **代码可读性**: 代码是否易于理解和维护
3. **测试完整性**: 测试是否覆盖关键场景
4. **文档完整性**: 文档是否同步更新
5. **性能影响**: 变更是否影响系统性能

## 常见问题检查

### 代码质量问题
- [ ] 过长的函数（>50行）
- [ ] 过深的嵌套（>3层）
- [ ] 魔术数字（应定义为常量）
- [ ] 重复代码（应考虑重构）
- [ ] 不清晰的变量名

### 64卦系统特定问题

- [ ] 卦象映射缺失或不准确

- [ ] 状态持久化失败
- [ ] 并发状态访问问题
