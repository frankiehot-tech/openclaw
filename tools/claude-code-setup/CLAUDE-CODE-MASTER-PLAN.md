### 3.3 指纹清理工具设计

```bash
#!/bin/bash
# clean-claude-fingerprints.sh
```

## 四、Phase 2：本地模型升级（解决"傻"问题）

### 4.1 目标
将本地模型从 Gemma 4 8B Q4 升级到 12B/26B 或 Qwen2.5-14B/32B，解决输出重复、浅层、格式混乱问题。

### 4.2 模型选型对比

| 模型 | 参数量 | 量化 | 质量 | 速度 | 内存 | 推荐度 |
|------|--------|------|------|------|------|--------|
| gemma4-claude (当前) | 8B | Q4 | ⭐⭐ | 快 | ~6GB | ❌ 淘汰 |
| gemma4:12b | 12B | Q4 | ⭐⭐⭐ | 快 | ~8GB | ⚠️ 备选 |
| gemma4:26b | 26B | Q4 | ⭐⭐⭐⭐ | 中 | ~16GB | ✅ 推荐 |
| qwen2.5:14b | 14B | Q4 | ⭐⭐⭐⭐ | 快 | ~9GB | ✅ 推荐 |
| qwen2.5:32b | 32B | Q4 | ⭐⭐⭐⭐⭐ | 中 | ~20GB | ✅ 最佳 |
| deepseek-r1:14b | 14B | Q4 | ⭐⭐⭐⭐⭐ | 中 | ~9GB | ✅ 推荐 |

### 4.3 推荐方案：Qwen2.5-14B（平衡选择）

**理由**：
- 14B 参数足以处理大多数编码任务
- Qwen2.5 架构在中文和代码场景表现优异
- ~9GB 内存占用，M4 24G 可轻松承载
- 速度快，延迟低

## 五、Phase 3：混合路由策略完善

### 5.1 目标
建立智能分层路由：本地处理轻量任务，云端处理重型任务，自动切换并提示成本。

### 5.2 路由决策树

```
用户输入
    │
    ├─→ 任务复杂度分析
    │       ├─→ 关键词匹配（分析/架构/重构/复杂）→ 重型
    │       ├─→ 文件大小 > 1000 行 → 重型
    │       ├─→ 上下文长度 > 16K tokens → 重型
    │       └─→ 其他 → 轻量
    │
    ├─→ 轻量任务 ──→ 本地 Ollama (Qwen2.5-14B)
    │                   零延迟、零成本
    │
    └─→ 重型任务 ──→ 云端 API
                        ├─→ 百炼 Qwen-Max（中文优先）
                        └─→ DeepSeek（推理优先）
```

### 5.3 重构 claude-dual-model.sh

```bash
#!/bin/bash
# 智能路由版本

# 任务复杂度判断函数
analyze_complexity() {
    local input="$1"
    
    # 重型任务关键词
    if echo "$input" | grep -qiE "(架构|重构|分析|设计|复杂|大型|深度|review|audit)"; then
        echo "heavy"
        return
    fi
    
    # 检查文件大小
    if [ -n "$CURRENT_FILE" ] && [ -f "$CURRENT_FILE" ]; then
        local lines=$(wc -l < "$CURRENT_FILE")
        if [ "$lines" -gt 1000 ]; then
            echo "heavy"
            return
        fi
    fi
    
    echo "light"
}

# 路由执行
route_task() {
    local complexity=$(analyze_complexity "$1")
    
    case "$complexity" in
        heavy)
            echo "🧠 重型任务 → 云端 Qwen-Max"
            setup_cloud_model "qwen-max"
            ;;
        light)
            echo "⚡ 轻量任务 → 本地 Qwen2.5-14B"
            setup_ollama_local "qwen2.5-claude"
            ;;
    esac
}
```

```python
# 在 dashscope-adapter.py 中添加
class FallbackManager:
    def __init__(self):
        self.primary = "dashscope"
        self.fallback = "deepseek"
        self.failure_count = 0
        self.threshold = 3
    
    def call(self, request):
        try:
            return self.call_primary(request)
        except Exception as e:
            self.failure_count += 1
            if self.failure_count >= self.threshold:
                logger.warning("主服务异常，切换到备用服务")
                return self.call_fallback(request)
            raise
```

### 7.2 评估维度

| 维度 | 权重 | 评估方法 |
|------|------|----------|
| 重复率 | 30% | 计算输出中重复句子比例 |
| 深度 | 25% | 是否提供具体技术细节而非泛泛而谈 |
| 准确性 | 25% | 代码是否正确，建议是否合理 |
| 格式规范 | 10% | 是否符合要求的输出格式 |
| 响应速度 | 10% | 首 token 延迟和总生成时间 |
