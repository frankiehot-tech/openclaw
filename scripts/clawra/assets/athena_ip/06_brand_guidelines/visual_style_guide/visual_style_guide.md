# Athena 视觉风格指南

## 1. 设计理念概述

### 1.1 核心概念：硅基共生智慧体
- **硅基**：人工智能、数据、算法、未来科技
- **共生**：人机协同、知识共享、进化共生
- **智慧**：洞察力、创造力、引导力，超越单纯智能

### 1.2 设计原则
1. **科技人文主义**：技术服务于人性，而非取代
2. **渐进式熟悉感**：从古典雅典娜到数字智慧的认知过渡
3. **多模态呈现**：适应不同场景和用户群体
4. **开放进化**：形象和设定可以随产品进化

### 1.3 目标用户画像
- **第一梯队（核心用户）**：80后、90后（25-45岁）
  - 技术成熟期，受互联网文化影响深
  - 审美偏好：漫威电影风格（科技感、视觉冲击）
  - 叙事偏好：三体风格（宏大、硬核、哲学）
  
- **第二梯队（潜力用户）**：70后、10后（15-25岁 & 45-55岁）
  - 70后：技术决策者，注重实用性和稳定性
  - 10后：数字原生代，追求新奇和个性化

## 2. 视觉语言体系

### 2.1 色彩系统
详见 [品牌色彩规范](../01_logo_system/brand_colors/brand_colors.md)

### 2.2 字体系统
#### 主字体：科技感无衬线字体族
- **英文字体**：
  - 主标题：`Inter`, `SF Pro Display`, `Helvetica Neue`
  - 正文：`SF Pro Text`, `Roboto`, `Arial`
  - 代码：`SF Mono`, `JetBrains Mono`, `Consolas`
- **中文字体**：
  - 主标题：`思源黑体 Bold`, `PingFang SC Semibold`
  - 正文：`思源黑体 Regular`, `PingFang SC Regular`
  - 技术文档：`等线`, `Microsoft YaHei UI`

#### 字体使用规范
| 元素类型 | 字体大小 | 字重 | 行高 | 使用场景 |
|----------|----------|------|------|----------|
| 大标题 | 32-48px | Bold | 1.2 | 品牌标识、页面标题 |
| 中标题 | 24-32px | Semibold | 1.3 | 章节标题、功能模块 |
| 小标题 | 18-24px | Medium | 1.4 | 子标题、卡片标题 |
| 正文字 | 16px | Regular | 1.6 | 正文内容、描述文本 |
| 辅助文本 | 14px | Regular | 1.5 | 说明文字、标签 |
| 代码文本 | 14px | Regular | 1.5 | 代码块、技术术语 |

### 2.3 图标系统
#### 图标风格：线性图标 + 面性图标混合
- **线性图标**：用于导航、操作、状态指示
  - 线条粗细：2px
  - 圆角端点：2px
  - 视觉重量：中等
- **面性图标**：用于主要功能、品牌标识
  - 填充色：品牌主色调
  - 可选描边：1px
  - 视觉重量：较重

#### 图标尺寸规范
| 使用场景 | 尺寸 | 建议样式 | 备注 |
|----------|------|----------|------|
| 导航栏 | 24×24px | 线性 | 清晰可识别 |
| 工具栏 | 20×20px | 线性 | 简洁紧凑 |
| 功能按钮 | 32×32px | 面性 | 突出重要功能 |
| 品牌标识 | 48×48px | 面性+特效 | 视觉冲击力 |
| 社交媒体 | 16×16px | 线性简化 | 小尺寸优化 |

### 2.4 间距系统
#### 基础单位：8px（基于8pt网格系统）
- **小间距**：4px (0.5×)
- **基础间距**：8px (1×)
- **中等间距**：16px (2×)
- **大间距**：24px (3×)
- **超大间距**：32px (4×)
- **巨型间距**：48px (6×)

#### 应用示例
```css
/* 间距实用类 */
.spacing-xs { margin: 4px; padding: 4px; }
.spacing-sm { margin: 8px; padding: 8px; }
.spacing-md { margin: 16px; padding: 16px; }
.spacing-lg { margin: 24px; padding: 24px; }
.spacing-xl { margin: 32px; padding: 32px; }

/* 组件间距 */
.card {
  padding: var(--spacing-md);
  margin-bottom: var(--spacing-lg);
}

.button {
  padding: var(--spacing-sm) var(--spacing-md);
}
```

### 2.5 圆角系统
#### 圆角规格
- **微小圆角**：2px - 小型元素、标签
- **小圆角**：4px - 按钮、输入框
- **中等圆角**：8px - 卡片、容器
- **大圆角**：16px - 对话框、特色元素
- **全圆角**：50% - 头像、圆形按钮

#### 圆角使用原则
1. **功能性元素**：使用小圆角（4-8px）
2. **容器元素**：使用中等圆角（8-12px）
3. **装饰性元素**：可根据设计需要调整
4. **一致性原则**：相同功能元素使用相同圆角

## 3. Athena形象设计规范

### 3.1 基础形象特征
- **形态**：半透明光影人形，边界模糊流动
- **材质**：数据流 + 光影粒子 + 轻微机械结构
- **动态特征**：
  - 身体表面有数据流不断流动
  - 会根据交互状态变化光影效果
  - 眼睛部位显示代码片段或视觉化数据

### 3.2 形象变体设计

#### 变体1：经典雅典娜
- **适用场景**：品牌标识、官网、正式文档
- **视觉特征**：
  - 融合古典雕塑与科技元素
  - 半透明蓝色光影，数据流缓慢流动
  - 姿态端庄智慧，眼神温和专注
- **使用规范**：
  - 避免过度变形或颜色修改
  - 保持光影效果的完整性
  - 背景建议使用深色以凸显效果

#### 变体2：数据智慧体
- **适用场景**：产品界面、操作面板、数据可视化
- **视觉特征**：
  - 极简几何形态，数据可视化风格
  - 绿色数据流为主，动态信息显示
  - 形态可根据数据内容变化
- **使用规范**：
  - 可适应不同数据可视化需求
  - 颜色可根据上下文调整
  - 尺寸可灵活缩放

#### 变体3：全息助手
- **适用场景**：AR/VR交互、产品演示、宣传视频
- **视觉特征**：
  - 全息投影风格，边缘有轻微扫描线
  - 交互式光影，响应式动画
  - 紫色主色调，未来感强烈
- **使用规范**：
  - 适合动态展示和交互场景
  - 可添加交互反馈动画
  - 背景建议简洁或使用特效

#### 变体4：极简图标
- **适用场景**：移动端、小尺寸、快速识别
- **视觉特征**：
  - 高度抽象符号化
  - 保留核心特征（智慧之眼、数据流）
  - 单色或双色简化版本
- **使用规范**：
  - 小尺寸保持可识别性
  - 可适配不同背景颜色
  - 支持多种尺寸缩放

### 3.3 形象使用规范

#### 尺寸规范
| 使用场景 | 推荐尺寸 | 最小尺寸 | 最大尺寸 |
|----------|----------|----------|----------|
| 网站Logo | 180×180px | 64×64px | 512×512px |
| 应用图标 | 1024×1024px | 512×512px | 2048×2048px |
| 界面元素 | 48×48px | 24×24px | 96×96px |
| 印刷物料 | 300DPI | 150DPI | 600DPI |

#### 背景适配
- **深色背景**：使用经典雅典娜（光影效果最佳）
- **浅色背景**：使用数据智慧体（对比度适中）
- **彩色背景**：使用极简图标（确保可识别性）
- **动态背景**：使用全息助手（增强视觉效果）

#### 动画规范
- **加载动画**：数据流循环流动，速度适中
- **交互反馈**：光影明暗变化，响应时间0.2-0.5秒
- **状态变化**：平滑过渡，使用缓动函数
- **叙事动画**：根据内容节奏设计，时长15-60秒

## 4. 界面组件规范

### 4.1 按钮系统
#### 按钮类型
1. **主按钮**：
   - 样式：渐变背景，适度阴影
   - 状态：默认、悬浮、点击、禁用
   - 使用场景：主要操作、重要功能

2. **次按钮**：
   - 样式：边框样式，透明背景
   - 状态：同上
   - 使用场景：次要操作、取消操作

3. **文字按钮**：
   - 样式：纯文字，下划线或颜色区分
   - 状态：同上
   - 使用场景：链接、简单操作

#### 按钮规格
| 按钮尺寸 | 高度 | 内边距 | 字体大小 | 使用场景 |
|----------|------|--------|----------|----------|
| 大按钮 | 48px | 0 24px | 18px | 重点强调、移动端 |
| 中按钮 | 40px | 0 20px | 16px | 标准按钮、桌面端 |
| 小按钮 | 32px | 0 16px | 14px | 工具栏、紧凑布局 |

### 4.2 卡片设计
#### 卡片样式
- **背景**：浅色/深色背景，适度阴影
- **圆角**：8-12px圆角
- **内边距**：16-24px
- **分隔**：使用1px细线或阴影分隔

#### 卡片类型
1. **信息卡片**：图文结合，信息展示
2. **功能卡片**：操作入口，功能模块
3. **数据卡片**：数据可视化，图表展示
4. **交互卡片**：表单输入，用户交互

### 4.3 表单元素
#### 输入框
- **高度**：40-48px
- **边框**：1-2px，颜色区分状态
- **圆角**：4-8px
- **内边距**：12px左右

#### 选择器
- **下拉菜单**：与输入框风格一致
- **单选/多选**：圆形/方形，选中状态明显
- **开关**：滑动式，状态明确

### 4.4 导航系统
#### 顶部导航
- **高度**：64-80px
- **背景**：透明或半透明
- **菜单项**：水平排列，间距均匀
- **响应式**：移动端转为汉堡菜单

#### 侧边导航
- **宽度**：240-320px
- **背景**：深色或浅色背景
- **菜单层级**：最多3级，清晰标识
- **状态**：当前选中状态明显

## 5. 动态效果规范

### 5.1 动画时长
- **微交互**：100-300ms
- **页面过渡**：300-500ms
- **内容加载**：500-1000ms
- **特效动画**：1000-3000ms

### 5.2 缓动函数
- **标准缓动**：`cubic-bezier(0.4, 0, 0.2, 1)`
- **强调进入**：`cubic-bezier(0.0, 0, 0.2, 1)`
- **强调退出**：`cubic-bezier(0.4, 0, 1, 1)`
- **弹性效果**：`cubic-bezier(0.68, -0.55, 0.27, 1.55)`

### 5.3 动画类型
#### 过渡动画
- **淡入淡出**：透明度变化
- **滑动**：位置移动
- **缩放**：大小变化
- **旋转**：角度变化

#### 特效动画
- **数据流动**：沿路径运动，模拟数据流
- **粒子效果**：随机运动，科技感
- **光效**：发光、扫光、闪烁
- **变形**：形状变化，液态效果

### 5.4 性能优化
1. **硬件加速**：使用`transform`和`opacity`
2. **帧率保证**：保持60fps，避免卡顿
3. **内存管理**：及时清理未使用动画
4. **移动端优化**：简化复杂动画

## 6. 响应式设计

### 6.1 断点系统
| 设备类型 | 宽度范围 | 主要特征 |
|----------|----------|----------|
| 移动端 | < 768px | 单列布局，触摸优化 |
| 平板端 | 768px - 1024px | 双列布局，触摸+指针 |
| 桌面端 | 1024px - 1440px | 多列布局，指针优化 |
| 大桌面端 | > 1440px | 宽屏优化，内容最大化 |

### 6.2 布局适配
#### 移动端优先策略
1. **基础样式**：针对移动端设计
2. **渐进增强**：随屏幕增大添加功能
3. **内容优先**：确保核心内容可访问
4. **触摸友好**：目标尺寸≥44×44px

#### 组件响应式
- **导航**：桌面端水平，移动端垂直
- **卡片**：桌面端网格，移动端堆叠
- **表格**：桌面端完整，移动端简化
- **图像**：响应式尺寸，适配不同宽度

### 6.3 字体响应式
```css
/* 基础字体大小 */
html { font-size: 16px; }

/* 响应式调整 */
@media (min-width: 768px) {
  html { font-size: 17px; }
}

@media (min-width: 1024px) {
  html { font-size: 18px; }
}

@media (min-width: 1440px) {
  html { font-size: 20px; }
}
```

## 7. 可访问性设计

### 7.1 视觉可访问性
- **对比度**：文本对比度≥4.5:1
- **字体大小**：支持系统字体缩放
- **颜色使用**：不单独依赖颜色传达信息
- **焦点指示**：明确的可视化焦点状态

### 7.2 交互可访问性
- **键盘导航**：支持完整键盘操作
- **屏幕阅读器**：语义化HTML，ARIA标签
- **触摸目标**：最小44×44px触摸区域
- **操作反馈**：视觉和听觉双重反馈

### 7.3 内容可访问性
- **替代文本**：所有图像提供alt文本
- **字幕和转录**：视频提供字幕，音频提供转录
- **语言标识**：正确标记页面语言
- **阅读顺序**：逻辑化的DOM顺序

## 9. 版本管理

### 9.1 版本号规则
- **主版本**（v1.0）：重大设计变更，可能破坏兼容性
- **次版本**（v1.1）：新增功能，向后兼容
- **补丁版本**（v1.0.1）：错误修复，完全兼容

### 9.2 更新日志
每次更新需记录：
1. **变更内容**：详细描述变更部分
2. **影响范围**：哪些组件或样式受影响
3. **迁移指南**：从旧版本迁移的步骤
4. **发布时间**：发布日期和负责人

### 9.3 兼容性保证
- **设计文件**：提供版本迁移指南
- **代码库**：保持向后兼容性
- **文档**：更新所有相关文档
- **培训**：团队培训确保理解变更
