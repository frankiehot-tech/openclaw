# MAREF状态空间扩展策略设计

## 设计背景
基于2026年4月14日的状态空间分析，当前系统只探索了5/64个可能状态（7.8%覆盖率）。为了提升系统的状态空间覆盖率，需要设计系统化的扩展策略。

## 当前状态空间分析

### 现有状态 (5个)
1. **000000** (坤为地) - 最常用状态 (31次，33.7%)
2. **000001** (地雷复) - 次常用状态 (23次，25.0%)
3. **000010** (水地比) - 常用状态 (16次，17.4%)
4. **000011** (泽地萃) - 常用状态 (15次，16.3%)
5. **000100** (山地剥) - 较少使用 (7次，7.6%)

### 现有转换模式 (6种)
1. 000000 → 000001 (15次)
2. 000001 → 000011 (8次)
3. 000011 → 000010 (7次)
4. 000010 → 000000 (7次)
5. 000000 → 000100 (7次)
6. 000000 → 000010 (2次)

### 状态空间拓扑特征
- 集中在状态空间低区域 (前5个低值状态)
- 转换模式较为简单，缺乏复杂性
- 状态访问频率极不均衡 (000000占比33.7%)

## 扩展目标

### 总体目标
将状态空间覆盖率从7.8%提升到至少25% (16/64个状态)，同时提高转换多样性。

### 具体目标
1. **数量目标**: 新增至少11个新状态，使总状态数达到16+
2. **分布目标**: 状态访问频率更加均衡
3. **转换目标**: 新增至少15种新的转换模式
4. **合规目标**: 保持100%格雷编码合规率

### 阶段目标
- **阶段1**: 扩展到12个状态 (18.75%覆盖率)
- **阶段2**: 扩展到16个状态 (25%覆盖率)
- **阶段3**: 扩展到20个状态 (31.25%覆盖率)

## 扩展策略设计

### 策略1: 邻居优先扩展 (基于汉明距离)

从现有每个状态出发，优先探索其汉明距离为1的邻居状态：

#### 现有状态的邻居分析
1. **000000** 的邻居 (改变1位):
   - ✅ 000001 (已覆盖)
   - ✅ 000010 (已覆盖)
   - ✅ 000100 (已覆盖)
   - 001000 (新)
   - 010000 (新)
   - 100000 (新)

2. **000001** 的邻居:
   - ✅ 000000 (已覆盖)
   - ✅ 000011 (已覆盖)
   - 000101 (新)
   - 001001 (新)
   - 010001 (新)
   - 100001 (新)

3. **000010** 的邻居:
   - ✅ 000000 (已覆盖)
   - ✅ 000011 (已覆盖)
   - 000110 (新)
   - 001010 (新)
   - 010010 (新)
   - 100010 (新)

4. **000011** 的邻居:
   - ✅ 000001 (已覆盖)
   - ✅ 000010 (已覆盖)
   - 000111 (新)
   - 001011 (新)
   - 010011 (新)
   - 100011 (新)

5. **000100** 的邻居:
   - ✅ 000000 (已覆盖)
   - 000101 (新)
   - 000110 (新)
   - 001100 (新)
   - 010100 (新)
   - 100100 (新)

#### 邻居扩展优先级
1. **一级优先级** (与多个现有状态相邻):
   - 000101 (与000001和000100相邻)
   - 000110 (与000010和000100相邻)
   - 001000 (仅与000000相邻，但位置重要)

2. **二级优先级** (与单个现有状态相邻):
   - 001001, 010001, 100001 (从000001)
   - 001010, 010010, 100010 (从000010)

3. **三级优先级** (填补空白区域):
   - 001100, 010100, 100100 (从000100)

### 策略2: 模式优先扩展

探索具有特殊模式的卦象，增加状态空间多样性：

#### 重要模式卦象
1. **交替模式**:
   - 010101 (火水未济) - 重要交替模式
   - 101010 (水火既济) - 互补交替模式

2. **对称模式**:
   - 001100 (艮为山) - 中心对称
   - 110011 (泽风大过) - 镜像对称

3. **边界模式**:
   - 111111 (乾为天) - 全阳爻，状态空间最大值
   - 000000 (已覆盖) - 全阴爻，状态空间最小值

4. **均匀分布模式**:
   - 每隔4个状态取一个样本，确保空间均匀性

### 策略3: 路径优先扩展

设计有意义的状态转换路径，增加转换多样性：

#### 路径类型设计
1. **线性扩展路径**: 沿着特定方向连续扩展
   - 示例: 000000 → 000001 → 000011 → 000111 → 001111 → 011111 → 111111

2. **星型扩展路径**: 从中心状态向多个方向扩展
   - 以000000为中心，向6个不同方向扩展

3. **循环扩展路径**: 形成闭环，验证系统稳定性
   - 示例: 000000 → 000001 → 000011 → 000010 → 000000

4. **深度扩展路径**: 在特定子空间中深入探索
   - 示例: 在低位三位固定的情况下探索高位三位变化
