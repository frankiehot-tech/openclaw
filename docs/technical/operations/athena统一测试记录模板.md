## 1. 统一测试记录主模板

### 测试记录头信息
- **记录ID**: `TEST-YYYYMMDD-HHMMSS`
- **模板版本**: `v1.0`
- **创建时间**: `YYYY-MM-DD HH:MM:SS`
- **测试环境**: `[设备型号/模拟器版本]`

## 2. 整理类任务专用记录字段

### 专用字段区（整理类任务必填）
| 字段名称 | 必填 | 填写说明 | 允许值 | 验证规则 |
|---------|------|---------|--------|----------|
| **notification_closed** | 是 | 通知栏是否已关闭 | `true` / `false` | ADB验证: `dumpsys window` 无NotificationShade焦点 |
| **launcher_confirmed** | 是 | 是否确认在launcher环境 | `true` / `false` | ADB验证: 当前焦点为launcher相关窗口 |
| **icon_grid_confirmed** | 是 | 视觉是否识别到图标网格 | `true` / `false` | 视觉分析: Qwen确认存在图标网格等桌面特征 |
| **page1_confirmed** | 是 | 是否确认在桌面第一页 | `true` / `false` | ADB+视觉双重验证一致，无状态冲突 |
| **drag_enabled** | 是 | 拖拽功能是否启用 | `true` / `false` | 仅在`page1_confirmed=true`时允许为`true` |

### 字段间依赖规则
1. **逐项必填规则**: 所有5个专用字段必须明确填写，不得为空
2. **拖拽启用条件**: `drag_enabled=true` **必须**满足 `page1_confirmed=true`
3. **状态一致性**: `page1_confirmed=true` **必须**满足 `notification_closed=true` ∧ `launcher_confirmed=true` ∧ `icon_grid_confirmed=true`
4. **冲突处理**: 当字段间逻辑矛盾时，记录自动标记为"状态冲突"失败类型

### 状态流转验证
```
初始状态 → Gate 6A验证 → 整理准备状态 → Gate 6B执行 → 完成状态
    ↓            ↓            ↓            ↓           ↓
notification   launcher     icon_grid   page1_confirmed drag_enabled
    ↓            ↓            ↓            ↓           ↓
[待验证] → [已关闭] → [已确认] → [已确认] → [已确认] → [启用/禁用]
```

## 3. 测试记录合格标准

### 3.1 记录完整性要求
- [ ] **必填字段完整**: 所有标"是"的字段必须填写
- [ ] **截图证据齐全**: 起始、每步、结束截图必须存在且可访问
- [ ] **时间序列正确**: 动作序列时间戳必须递增且合理
- [ ] **状态判断一致**: 前后状态判断逻辑自洽

### 3.2 整理类任务特殊要求
- [ ] **专用字段完整**: 5个专用字段全部填写
- [ ] **依赖规则遵守**: `page1_confirmed≠true`时`drag_enabled≠true`
- [ ] **双重验证记录**: ADB和视觉验证依据分别记录
- [ ] **冲突处理记录**: 状态冲突时的详细处理过程

### 3.3 测试记录不合格情形（任意一项即不合格）

#### 3.3.1 基础完整性不合格
1. **必填字段缺失**: 任意必填字段为空
2. **截图证据缺失**: 缺少起始、结束或关键步骤截图
3. **时间逻辑错误**: 动作时间序列混乱或倒置
4. **状态判断矛盾**: 前后状态描述明显矛盾

#### 3.3.2 执行过程不合格
5. **安全规则违反**: 未按安全规则停止（如状态冲突仍继续）
6. **验证机制跳过**: 跳过ADB或视觉任一验证环节
7. **硬规则违反**: 违反Gate 6A/6B硬规则（如未确认桌面执行拖拽）
8. **超规动作执行**: 执行规范明确禁止的动作

#### 3.3.3 整理类专用不合格
9. **专用字段缺失**: 5个专用字段任一项未填写
10. **依赖规则违反**: `page1_confirmed=false`时`drag_enabled=true`
11. **状态冲突未处理**: ADB与视觉验证冲突未记录和处理
12. **双重验证缺失**: 仅凭单一验证方式确认状态

#### 3.3.4 证据链不合格
13. **截图无法验证**: 截图模糊、黑屏或无法识别状态
14. **验证依据缺失**: 状态判断无具体依据支持
15. **执行命令错误**: 记录的命令与实际执行不符
16. **时间戳异常**: 执行时间超出合理范围

### 3.4 模板用途说明

本模板专用于以下四大场景：

#### 3.4.1 回归测试
- **目的**: 确保Gate 6A/6B规范修改后不影响现有功能
- **使用方式**: 每次规范更新后，使用本模板执行完整测试流程
- **验收标准**: 所有历史测试用例仍能通过

#### 3.4.2 失败统计
- **目的**: 系统性分析失败原因，识别高危失败模式
- **使用方式**: 失败测试必须填写完整，包括失败类型和原因分析
- **统计维度**: 按失败类型、设备型号、时间周期聚合分析

#### 3.4.3 安全复盘
- **目的**: 审计高风险动作，确保安全规则有效执行
- **使用方式**: 重点关注"是否安全停止"和状态冲突处理
- **复盘重点**: 检查安全边界是否被突破，规则是否完善

#### 3.4.4 高风险动作审计
- **目的**: 监控拖拽等高风险动作的执行合规性
- **使用方式**: 整理类任务必须详细记录拖拽启用条件和执行过程
- **审计要点**: `drag_enabled`启用条件是否严格，动作是否保守

### 3.5 记录保存与追溯要求
1. **永久保存**: 所有测试记录必须永久保存，不得删除
2. **版本关联**: 记录必须关联测试时的代码版本和环境版本
3. **可追溯性**: 通过记录ID可追溯完整的执行链
4. **定期审计**: 每月对测试记录进行合规性审计
