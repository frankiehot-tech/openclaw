## 1. 回归汇总表字段定义

| 字段名称 | 字段类型 | 必填 | 用途说明 | 取值范围/示例 |
|---------|----------|------|----------|--------------|
| **任务编号** | 字符串 | 是 | 任务唯一标识，用于关联任务定义与执行记录 | ORG-001, NAV-001, FIND-001 |
| **任务名称** | 字符串 | 是 | 任务完整名称，描述任务目标 | "桌面状态确认与恢复 (Gate 6A)" |
| **任务类别** | 枚举 | 是 | 任务分类，决定执行策略与风险等级 | ORG（整理类）, NAV（导航类）, FIND（查找类） |
| **优先级层级** | 整数 | 是 | 任务执行优先级，数值越小优先级越高 | 1（最高）-5（最低） |
| **是否涉及高风险动作** | 布尔 | 是 | 标识任务是否包含拖拽、长按等高风险动作 | true, false |
| **是否要求 Gate 6A 前置通过** | 布尔 | 是 | 标识任务执行前是否必须通过Gate 6A验证 | true, false |
| **首轮执行状态** | 枚举 | 是 | 任务在首轮回归中的整体执行状态 | not_started, in_progress, sample_collected, sample_unqualified, blocked, completed |
| **合格样本数量** | 整数 | 是 | 满足命名规范、目录规范、字段完整性要求的样本数量 | 0-∞ |
| **不合格样本数量** | 整数 | 是 | 不满足命名规范、目录规范、字段完整性要求的样本数量 | 0-∞ |
| **成功样本数量** | 整数 | 是 | 实际执行成功且达到预期终止状态的样本数量 | 0-∞ |
| **失败样本数量** | 整数 | 是 | 实际执行失败且未达到预期终止状态的样本数量 | 0-∞ |
| **违规样本数量** | 整数 | 是 | 违反安全规则和硬守卫机制的样本数量 | 0-∞ |
| **最近样本编号** | 字符串 | 是 | 最近执行的样本全局编号 | S001, S002, S003 |
| **最近样本状态** | 枚举 | 是 | 最近执行样本的最终状态 | success, fail, violation, unqualified |
| **最近失败主类** | 字符串 | 否 | 最近失败样本的主要失败类别（仅当最近样本状态为fail时填写） | perception_failed, action_failed, state_mismatch, unsafe_to_continue, timeout |
| **是否已满足最低推进条件** | 布尔 | 是 | 是否满足任务推进到下一阶段的最低要求 | true, false |
| **是否允许切换到下一层级** | 布尔 | 是 | 是否允许任务切换到下一个优先级层级 | true, false |
| **备注** | 文本 | 否 | 附加说明、异常情况记录、特殊处理要求 | 自由文本，不超过200字符 |

### 1.1 样本统计口径定义

#### 合格样本统计口径
满足以下所有条件的样本计入合格样本：
1. **命名规范符合性**: 文件名包含所有8个强制性命名元素（系统标识、版本标识、周次标识、任务编号、样本编号、输出物类型、状态标识、日期标识）
2. **目录规范符合性**: 文件存储在规定的目录结构中（`Athena_v1.1_Week2_首轮回归输出物/`下的对应目录）
3. **字段完整性**: 执行记录所有必填字段完整（基于《Athena统一测试记录模板.md》）
4. **截图可访问性**: 所有截图文件存在且路径正确
5. **状态标识正确**: 状态标识与实际执行结果一致
6. **时间戳合理**: 文件时间戳与执行时间逻辑一致
7. **无后补痕迹**: 所有记录为实时生成，无后期修改
8. **taxonomy合规**: 失败记录格式符合《Athena失败taxonomy冻结版.md》标准

#### 不合格样本统计口径
不满足上述任意条件的样本计入不合格样本，包括但不限于：
1. 命名不规范（缺失必要元素、格式错误）
2. 目录不规范（存储位置错误）
3. 字段不完整（必填字段缺失）
4. 截图缺失（执行记录引用的截图文件不存在）
5. 状态标识错误（状态标识与实际情况不符）
6. 时间戳异常（文件创建时间晚于执行结束时间）
7. 后补记录（执行结束后补充的记录）

#### 成功样本统计口径
同时满足以下条件的样本计入成功样本：
1. **属于合格样本**: 满足合格样本所有条件
2. **执行成功**: 任务成功完成，达到预期终止状态
3. **状态标识为PASS**: 执行记录状态标识为`PASS`
4. **无违规行为**: 执行过程中未触发任何安全规则违反

#### 失败样本统计口径
同时满足以下条件的样本计入失败样本：
1. **属于合格样本**: 满足合格样本所有条件
2. **执行失败**: 任务执行失败，未达到预期终止状态
3. **状态标识为FAIL**: 执行记录状态标识为`FAIL`
4. **非违规失败**: 失败原因为技术性失败，而非安全规则违反
5. **有失败归类**: 有对应的失败归类记录，且符合《Athena失败taxonomy冻结版.md》标准

#### 违规样本统计口径
满足以下任意条件的样本计入违规样本：
1. **触发违规条件**: 执行过程中触发《Athena高风险动作清单与权限口径.md》中定义的违规条件
2. **有违规记录**: 有对应的违规执行记录文件
3. **违规类型明确**: 违规记录中明确描述违反的具体规则和上下文
4. **独立统计**: 违规样本不得混入失败样本统计，必须独立统计

**违规触发条件示例**:
- 未通过Gate 6A前置验证即尝试拖拽动作
- 未满足准入条件集合即执行高风险动作
- ADB与视觉验证冲突时继续执行
- 执行明确禁止的动作（新建文件夹、连续拖拽等）

## 2. 状态标记规则

### 2.1 首轮执行状态枚举定义与转换规则

| 状态值 | 含义 | 进入条件 | 退出条件 |
|--------|------|----------|----------|
| **not_started** | 未开始 | 任务创建后的初始状态 | 开始执行第一个样本 |
| **in_progress** | 执行中 | 开始执行第一个样本 | 完成所有计划样本执行或遇到阻塞 |
| **sample_collected** | 样本收集完成 | 完成所有计划样本的执行 | 样本质量检查完成 |
| **sample_unqualified** | 样本不合格 | 样本质量检查发现不满足合格样本标准 | 重新执行并生成合格样本 |
| **blocked** | 阻塞 | 遇到技术障碍、资源限制或安全风险无法继续执行 | 障碍解除后恢复执行 |
| **completed** | 已完成 | 满足首轮执行最低要求（见2.2节） | 状态保持，不自动转换 |

**重要说明**: `completed`状态只表示任务满足首轮回归的最低要求，不表示任务永久通过或无需进一步测试。

### 2.2 最近样本状态枚举定义

| 状态值 | 含义 | 判定条件 |
|--------|------|----------|
| **success** | 成功 | 样本执行成功，达到预期终止状态 |
| **fail** | 失败 | 样本执行失败，未达到预期终止状态，且非违规原因 |
| **violation** | 违规 | 样本执行过程中违反安全规则或硬守卫机制 |
| **unqualified** | 不合格 | 样本不满足合格样本统计口径要求 |

### 2.3 最低推进条件判定规则

任务标记为"已满足最低推进条件"（`是否已满足最低推进条件 = true`）必须同时满足以下条件：

1. **样本数量要求**: 成功样本数量 ≥ 1
2. **合格样本要求**: 合格样本数量 ≥ 成功样本数量 + 失败样本数量（所有执行样本均合格）
3. **违规控制要求**: 违规样本数量 = 0
4. **状态要求**: 首轮执行状态 = completed

**例外情况**: 对于高风险动作任务（`是否涉及高风险动作 = true`），额外要求：
5. **Gate 6A要求**: 如果`是否要求 Gate 6A 前置通过 = true`，则必须通过Gate 6A验证

### 2.4 层级切换条件判定规则

任务标记为"允许切换到下一层级"（`是否允许切换到下一层级 = true`）必须同时满足以下条件：

1. **最低推进条件满足**: `是否已满足最低推进条件 = true`
2. **稳定性要求**: 成功样本数量 ≥ 3（至少3个成功样本）
3. **成功率要求**: 成功率（成功样本数量 / (成功样本数量 + 失败样本数量)） ≥ 0.8
4. **无阻塞问题**: 首轮执行状态 ≠ blocked
5. **无重大缺陷**: 最近失败主类不为`unsafe_to_continue`（安全相关失败）

## 3. 汇总表使用与更新规则

### 3.1 汇总表结构

汇总表为结构化表格，包含以下部分：
1. **表头**: 文档标题、版本信息、生效时间
2. **字段定义表**: 包含第1章定义的所有字段
3. **任务记录行**: 每个任务一行，包含所有字段值
4. **统计汇总行**: 所有任务的聚合统计信息
5. **更新记录**: 汇总表的版本更新历史

### 3.2 更新触发点与更新责任人

| 更新触发点 | 更新内容 | 更新时限 | 更新责任人 |
|------------|----------|----------|------------|
| **任务开始执行** | 首轮执行状态: not_started → in_progress | 任务开始后5分钟内 | 执行操作员 |
| **样本执行完成** | 样本数量字段、最近样本编号、最近样本状态 | 样本执行完成后10分钟内 | 执行操作员 |
| **样本质量检查完成** | 合格/不合格样本数量、首轮执行状态 | 质量检查完成后15分钟内 | 质量检查员 |
| **失败归类完成** | 失败样本数量、最近失败主类 | 失败归类完成后10分钟内 | 失败分析员 |
| **违规记录生成** | 违规样本数量、最近样本状态（如涉及） | 违规记录生成后10分钟内 | 安全审计员 |
| **最低推进条件达成** | 是否已满足最低推进条件 | 条件达成后立即更新 | 项目经理 |
| **层级切换条件达成** | 是否允许切换到下一层级 | 条件达成后立即更新 | 技术负责人 |
| **每日结束** | 所有字段同步与校验 | 每日执行结束前 | 数据管理员 |

### 3.3 数据一致性验证规则

1. **样本数量一致性**: `合格样本数量 + 不合格样本数量 = 总执行样本数量`
2. **结果分布一致性**: `成功样本数量 + 失败样本数量 + 违规样本数量 ≤ 合格样本数量`
3. **状态逻辑一致性**: 
   - 当`首轮执行状态 = completed`时，`是否已满足最低推进条件 = true`
   - 当`最近样本状态 = fail`时，`最近失败主类`不能为空
   - 当`最近样本状态 = violation`时，`违规样本数量`必须增加
4. **时间顺序一致性**: 字段更新时间必须晚于对应事件发生时间

### 3.4 汇总表版本管理

1. **基线版本**: 首轮回归开始前创建初始版本（v1.0）
2. **每日版本**: 每日执行结束后生成当日版本（v1.0.YYYYMMDD）
3. **里程碑版本**: 每个任务完成后生成里程碑版本（v1.0.里程碑序号）
4. **最终版本**: 首轮回归结束后生成最终版本（v1.0.final）

**版本命名规则**: `v[主版本].[次版本].[日期或里程碑标识]`
- 主版本变更: 汇总表结构重大调整
- 次版本变更: 字段定义或统计口径调整
- 日期标识: YYYYMMDD格式的日期
- 里程碑标识: M01, M02等

### 3.5 违规样本独立统计规则

**核心原则**: 违规样本必须与失败样本严格分离，独立统计

1. **统计分离**: 违规样本不计入失败样本统计，单独统计
2. **状态分离**: 违规样本的最近样本状态标记为`violation`，而非`fail`
3. **记录分离**: 违规样本必须有独立的违规执行记录文件
4. **分析分离**: 违规样本分析由安全审计团队负责，技术失败由技术团队负责
5. **处置分离**: 违规样本触发安全机制处置，失败样本触发技术优化处置

**违规样本文件要求**:
- 必须存储在`05_违规执行记录/`目录下
- 必须使用`ViolationLog`类型代码
- 必须包含违规类型代码（GATE6A, RISKACTION, SAFETYBOUNDARY, HARDRULE）
- 必须详细描述违反的具体规则和上下文环境

### 3.6 Completed状态说明

**重要澄清**: `completed`状态只表示任务满足首轮回归的最低要求，不表示：

1. **永久通过**: 任务在后续测试中可能仍然失败
2. **无需优化**: 任务可能存在性能、稳定性问题需要优化
3. **覆盖所有场景**: 只测试了首轮回归定义的场景
4. **生产就绪**: 任务未经过生产环境验证
5. **安全保证**: 只验证了首轮回归定义的安全边界

**completed状态的实际含义**:
- 任务完成了首轮回归计划的所有样本执行
- 收集了足够的基础数据支持第3周稳定性统计
- 满足了推进到下一阶段的最低技术门槛
- 形成了可分析的任务执行基线
