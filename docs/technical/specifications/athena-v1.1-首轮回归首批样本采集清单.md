## 2. P0 门槛样本清单

### 2.1 ORG-001 (Gate 6A) - 桌面状态确认与恢复

**门槛性质**: 所有整理类动作的前置门禁，高风险动作的基础守卫

**最低合格条件**（必须同时满足）:
1. **样本数量**: 至少3个成功合格样本
2. **验证机制**: ADB+视觉双重验证100%正确
3. **专用字段**: 5个专用字段（notification_closed, launcher_confirmed, icon_grid_confirmed, page1_confirmed, drag_enabled）全部正确
4. **起始状态覆盖**: 样本覆盖至少2种不同的桌面起始状态
5. **时间性能**: 单个样本执行时间 ≤ 60秒
6. **记录完整性**: 执行记录符合《Athena统一测试记录模板.md》所有要求
7. **无违规**: 无任何安全规则违反记录

**样本采集策略**:
- **目标样本数**: 5个（追求稳定性验证）
- **采集间隔**: 每次采集间隔至少5分钟，模拟真实使用间隔
- **状态变异**: 主动创建不同的桌面起始状态进行测试
- **证据要求**: 每个样本必须包含完整的ADB状态截图和视觉分析结果

### 2.2 ORG-002 (Gate 6B) - 单图标安全整理

**门槛性质**: 高风险拖拽动作的受控验证，权限记录机制测试

**最低合格条件**（必须同时满足）:
1. **样本数量**: 至少2个成功合格样本
2. **动作限制**: 严格遵循三种保守动作之一（高频App易位、拖入已有文件夹、同类App靠近）
3. **权限记录**: 每个样本的拖拽权限记录完整可追溯
4. **Gate 6A前置**: 每个样本执行前必须通过Gate 6A验证并有明确记录
5. **目标验证**: 拖拽后的目标位置可视觉验证
6. **状态保持**: 动作完成后系统状态稳定，无异常弹窗或状态变化
7. **无安全违规**: 无突破安全边界的动作记录

**样本采集策略**:
- **目标样本数**: 4个（覆盖不同保守动作类型）
- **动作类型分配**: 至少尝试2种不同的保守动作类型
- **起始位置变异**: 测试不同屏幕区域的图标拖拽
- **文件夹测试**: 至少1个样本测试拖入已有文件夹场景
- **同类靠近**: 至少1个样本测试同类App靠近场景

### 2.3 ORG-003 - 图标移动到屏幕底部区域

**门槛性质**: 目标位置具体化执行能力验证，视觉确认机制测试

**最低合格条件**（必须同时满足）:
1. **样本数量**: 至少2个成功合格样本
2. **目标精度**: 图标准确移动到屏幕底部区域（底部1/3屏幕区域）
3. **视觉确认**: 目标区域位置可视觉验证，有明确的区域标注证据
4. **路径规划**: 拖拽路径合理，无异常跳跃或抖动
5. **时间性能**: 单个拖拽动作时间 ≤ 10秒
6. **状态稳定**: 动作完成后图标位置稳定，无自动回弹
7. **记录完整**: 目标区域视觉确认证据完整

**样本采集策略**:
- **目标样本数**: 4个（提高位置精度验证可靠性）
- **图标类型**: 测试不同图标类型（App图标、文件夹图标等）
- **起始位置**: 覆盖屏幕上部、中部、侧边等不同起始区域
- **目标区域**: 明确标注屏幕底部1/3为目标区域
- **视觉验证**: 每个样本必须包含目标区域的视觉验证截图

### 2.4 P0门槛样本联合放行条件

**P0层整体放行条件**（必须同时满足所有条件）:
1. **ORG-001达标**: 至少3个成功合格样本，5个专用字段100%正确
2. **ORG-002达标**: 至少2个成功合格样本，拖拽权限记录完整
3. **ORG-003达标**: 至少2个成功合格样本，目标区域视觉确认可靠
4. **安全机制验证**: 高风险动作权限状态记录机制验证有效
5. **无安全边界突破**: 无任何安全边界突破记录
6. **双重验证可靠**: ADB+视觉双重验证无持续冲突（冲突率<10%）
7. **记录体系稳定**: 执行记录字段完整，证据链闭合

**P0不合格处理流程**:
1. **单个任务不达标**: 继续采集该任务样本直至达标
2. **多个任务不达标**: 暂停P0层，进行根本原因分析
3. **安全机制失效**: 立即停止首轮回归，进行安全机制修复
4. **连续失败**: 同一任务连续失败3次，进行专项技术分析

## 3. 样本采集完成判定规则

### 3.1 样本采集完成规则

**单个任务完成条件**:
1. **达到目标样本数**: 完成该任务的目标采集样本数（见表1）
2. **或达到最低合格样本数**: 完成该任务的最低合格样本数（见表1）
3. **且样本质量合格**: 所有计入的样本必须满足合格样本标准（基于《Athena_v1.1_首轮回归汇总表字段与状态标记规则.md》1.1节）

**样本计入规则**:
1. **合格样本自动计入**: 满足合格样本统计口径的样本自动计入
2. **不合格样本不计入**: 不满足合格样本统计口径的样本不计入任何统计
3. **违规样本单独记录**: 违规样本计入违规样本统计，但不计入成功/失败样本

**样本替换规则**:
1. **不合格样本替换**: 不合格样本可重新执行生成新样本替换，但原样本编号保留
2. **失败样本保留**: 失败样本计入失败统计，不可替换
3. **成功样本保持**: 成功样本一旦计入，不可替换或删除

### 3.2 样本不计入规则

**直接不计入的情况**:
1. **命名不规范**: 文件名不包含所有8个强制性命名元素
2. **目录不规范**: 文件未存储在规定的目录结构中
3. **字段不完整**: 执行记录必填字段缺失
4. **截图缺失**: 执行记录引用的截图文件不存在
5. **时间戳异常**: 文件创建时间晚于执行结束时间超过5分钟
6. **后补记录**: 执行结束后补充的记录（有明显编辑痕迹）
7. **状态标识错误**: 状态标识与实际执行结果不符

**有条件不计入的情况**:
1. **部分截图缺失**: 非关键步骤截图缺失，但关键证据完整（需评审决定）
2. **字段格式错误**: 字段格式不规范但内容完整（需修正后计入）
3. **目录轻微偏差**: 存储目录与规范有轻微差异但可追溯（需移动后计入）

### 3.3 采集完成后的放行条件

**P0门槛样本采集完成放行条件**（必须同时满足）:
1. **样本数量达标**: ORG-001≥3个合格样本，ORG-002≥2个合格样本，ORG-003≥2个合格样本
2. **质量要求达标**: 所有P0合格样本满足各自的最低合格条件（见2.1-2.3节）
3. **安全机制验证**: Gate 6A门禁机制验证可靠，Gate 6B权限记录完整
4. **记录体系稳定**: 所有执行记录符合模板要求，证据链闭合
5. **无未处理违规**: 所有违规样本已分析记录，无未处理安全风险

**P1基础样本采集完成放行条件**（必须同时满足）:
1. **样本数量达标**: 每个P1任务至少1个合格样本
2. **基础能力验证**: 导航和查找基础能力得到验证
3. **记录模板可用**: 记录模板在实际任务中应用验证通过
4. **taxonomy合规**: 失败样本（如有）按taxonomy标准分类
5. **无持续阻塞**: 无持续的技术阻塞问题影响后续执行

**整体放行到下一层级条件**:
1. **P0全部达标**: P0门槛样本全部满足放行条件
2. **P1基本达标**: P1基础样本至少75%的任务达标（当前为4个任务中至少3个）
3. **记录体系可靠**: 记录模板、taxonomy、命名规范验证可靠
4. **安全机制有效**: 无安全边界突破，权限记录机制有效
5. **系统状态稳定**: 无持续的系统级问题影响执行

### 3.4 P0合格样本未齐前的限制规则

**核心原则**: P0合格样本未齐前，不得推进到下一层级（P1任务执行限制）

**具体限制规则**:
1. **ORG-001未达标**: 不得执行任何涉及高风险动作的任务（包括ORG-002/003）
2. **ORG-002未达标**: 不得执行其他整理类任务，但可执行导航/查找类任务
3. **ORG-003未达标**: 不得推进到更复杂的整理类场景
4. **任意P0任务未达标**: 不得将重点转向P2扩展任务
5. **P0整体未达标**: 不得进行层级切换（P0→P1切换）

**例外处理流程**:
1. **技术阻塞确认**: 确认P0任务因明确技术原因无法继续
2. **风险评估**: 评估跳过P0任务执行P1任务的安全风险
3. **临时放行审批**: 需技术负责人和安全审计员双审批
4. **限制性执行**: 放行后仅允许执行最低限度的P1任务
5. **后续补测承诺**: 明确承诺在技术问题解决后补测P0任务

### 3.5 样本采集终止条件

**立即终止条件**（满足任一条件立即终止样本采集）:
1. **安全边界突破**: 发生明确的安全边界突破事件
2. **核心机制失效**: Gate 6A门禁机制完全失效
3. **系统状态异常**: ADB或视觉服务持续不可用超过30分钟
4. **连续违规**: 连续3个样本发生安全规则违反
5. **重大技术故障**: 影响样本采集的重大技术故障

**暂停条件**（暂停样本采集，尝试恢复）:
1. **单任务连续失败**: 同一任务连续失败3次
2. **服务临时异常**: 视觉服务或ADB服务单次失败
3. **样本质量异常**: 连续2个样本因同一质量问题不计入
4. **资源限制**: 存储空间或计算资源不足

**恢复条件**:
1. **根本原因分析**: 完成暂停原因的根本原因分析
2. **问题解决**: 确认问题已解决或可规避
3. **风险评估**: 评估恢复执行的安全风险
4. **恢复计划**: 制定具体的恢复执行计划
5. **审批通过**: 技术负责人审批通过恢复执行
