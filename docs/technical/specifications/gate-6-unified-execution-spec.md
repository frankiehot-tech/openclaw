# Gate 6A / Gate 6B 统一执行规范

## 1. Gate 6A：桌面恢复与确认规范

### 1.1 定义
**Gate 6A**：从任意状态恢复到桌面第一页，并确认允许进入整理动作。

### 1.2 核心概念区分
- **桌面态**：泛指设备处于launcher环境，可能包括通知栏展开、锁屏、应用覆盖等非纯净状态。
- **桌面第一页**：特指launcher的主屏幕第一页，无通知栏、无应用覆盖、无系统弹窗的纯净可整理状态。

### 1.3 通过条件（必须全部满足）
1. **通知栏关闭**：ADB确认无NotificationShade窗口焦点。
2. **已回到 launcher**：ADB确认当前焦点为launcher相关窗口。
3. **视觉能识别桌面图标网格**：Qwen视觉分析确认存在图标网格、大Widget、文件夹或搜索栏等桌面特征。
4. **ADB 与视觉不冲突**：ADB状态与视觉分析结果一致，无状态冲突。
5. **未执行任何无效拖拽**：本次Gate执行期间未触发任何拖拽动作。

### 1.4 硬规则
1. **未确认桌面第一页前，禁止任何拖拽动作**：状态不确定时，拖拽功能默认禁用。
2. **双重验证机制**：必须同时通过ADB验证（`dumpsys window`）和视觉验证（Qwen分析）。
3. **冲突即停止**：ADB与视觉状态冲突时，立即安全停止，不尝试恢复。
4. **保守恢复策略**：优先使用系统级操作（主页键、返回键），避免模拟点击。

### 1.5 停止条件
1. **无法恢复到桌面**：经过最大尝试次数（默认5次）后仍无法确认桌面第一页。
2. **状态冲突**：ADB与视觉验证结果不一致。
3. **截图失败**：无法获取当前屏幕状态。
4. **服务异常**：Qwen视觉服务不可用。

## 2. Gate 6B：单图标保守整理规范

### 2.1 定义
**Gate 6B**：仅在 Gate 6A 已通过前提下执行一次单图标保守整理。

### 2.2 适用边界
**允许动作（仅限以下三种之一）**：
1. **高频App易位**：将1个高频App移到更易访问位置（屏幕底部、中部）。
2. **拖入已有文件夹**：将1个App拖入已有文件夹（必须视觉确认文件夹存在）。
3. **同类App靠近**：将2个明显同类App靠近（如同是社交、工具、游戏类）。

### 2.3 禁止范围
1. **新建文件夹**：禁止任何创建新文件夹的操作。
2. **连续拖拽**：禁止连续拖拽多个图标。
3. **跨页整理**：禁止将图标拖到其他桌面页面。
4. **未确认状态拖拽**：桌面状态未通过Gate 6A验证时，禁止执行拖拽。
5. **高风险动作**：禁止删除图标、卸载应用、修改系统设置。

### 2.4 执行要求
1. **硬守卫前置**：必须通过Gate 6A验证，获取"允许整理"状态。
2. **单动作限制**：仅执行1个整理动作，完成后立即停止。
3. **位置保守**：拖拽起止点必须在安全区域内（避开状态栏、导航条、Widget）。
4. **效果验证**：动作前后截图对比，确认桌面状态变化。
5. **状态保持**：整理后必须验证仍在桌面第一页。

### 2.5 失败处理
1. **拖拽失败**：ADB swipe命令执行失败时，立即停止，不重试。
2. **状态丢失**：整理后离开桌面第一页，记录警告，停止后续动作。
3. **无合适动作**：视觉分析未找到合适整理机会时，使用保守测试动作（屏幕中部短距离移动）。

## 3. 整理类任务统一准入规则

### 3.1 核心原则
**所有整理动作必须先过 Gate 6A，拖拽默认禁用，无法确认即停止。**

### 3.2 准入流程
```
开始整理任务
    ↓
[Gate 6A] 桌面恢复与确认
    ├── 成功 → 解锁拖拽功能，进入[Gate 6B]
    └── 失败 → 安全停止，记录状态不确认
    ↓
[Gate 6B] 单图标保守整理（仅当Gate 6A通过）
    ├── 成功 → 完成1个整理动作，停止
    └── 失败 → 安全停止，记录动作失败
```

### 3.3 拖拽禁用规则
1. **默认状态**：系统启动时，拖拽功能默认为禁用状态。
2. **解锁条件**：仅当Gate 6A返回"桌面第一页确认通过"时，临时解锁拖拽功能。
3. **单次有效**：拖拽解锁仅对紧接着的1个整理动作有效，动作完成后立即重新禁用。
4. **状态丢失重锁**：任何时间点检测到离开桌面第一页，立即重新禁用拖拽功能。

### 3.4 无法确认即停止规则
1. **视觉不确定**：Qwen无法明确判断是否为桌面时，停止。
2. **ADB不确定**：`dumpsys window`无法确认launcher焦点时，停止。
3. **状态冲突**：ADB与视觉结果不一致时，停止。
4. **超时未确认**：恢复桌面尝试超过最大次数仍未确认时，停止。
5. **服务异常**：关键服务（ADB、Qwen）不可用时，停止。

### 3.5 安全边界
1. **动作隔离**：每个整理动作独立验证，不依赖前后动作状态。
2. **无状态记忆**：不记忆之前的成功状态，每次整理都重新验证Gate 6A。
3. **无自动重试**：失败后不自动重试，需要外部触发重新验证。
4. **无推测执行**：不推测用户意图，不执行未经明确验证的动作。
