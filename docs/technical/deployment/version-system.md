# Karpathy AutoResearch 版本体系说明

> 解决压力测试发现的 "三轨制版本共存但未文档化" 问题 (D3-1)

## 三轨版本体系

Karpathy AutoResearch 系统存在三种独立的版本体系，分别管辖不同层级的工件：

| 版本体系 | 当前版本 | 管辖范围 | 示例工件 |
|---------|--------|---------|---------|
| **项目版本** | v0.2.0 | 整体项目迭代阶段的里程碑 | 已批准方案文档、消费路径分析、AR-001到AR-006方案编号 |
| **研究文档版本** | v1.0.0 | 研究交付物（报告、白皮书）的语义版本 | 综合研究总报告、全链路审计报告、Skill 部署报告 |
| **Skill 版本** | v2.0.0 | 单个 Skill 工件的独立语义版本 | karpathy-principles.json (v2.0.0)、各 .md Skill |

## 为什么需要三轨制

```
项目迭代 (v0.x) : 回答 "这个方案是第几轮迭代的？"
  └─ 被消费的文档、方案编号属于此体系

研究文档 (v1.x) : 回答 "这篇研究报告经过多少次修订？"
  └─ 方法论变化、纠错、补丁属于此体系

Skill 工件 (v2.x) : 回答 "这个 Skill 的协议版本是什么？"
  └─ Skill 格式、协议、触发链版本独立于此体系
```

三轨分离的必要性：
1. **项目版本** 是时间线版本 — 随 MVP 迭代推进（v0.1 → v0.2 → v0.3）
2. **研究文档版本** 是内容版本 — 研究报告可能在项目迭代周期内多次修订
3. **Skill 版本** 是协议版本 — Skill 格式和触发机制独立于研究内容变化

## 版本切换规则

| 场景 | 何时升级版本 |
|------|------------|
| 项目版本 (v0.x) | 新轮次方案获批、邮件归入新归档目录 |
| 研究文档版本 (v1.x) | 重大修订（结构变化、方法论更新）；小幅修订用 v1.0.1 |
| Skill 版本 (v2.x) | 协议/触发链/格式变化；内容修补不升级版本 |

## 版本溯源

可通过以下字段追溯各工件的版本归属：

```
项目版本 → 文档路径中的 v0.2.0 目录
研究文档版本 → 文档头部 version 字段
Skill 版本 → YAML frontmatter 中的 version 字段 (或 JSON 中的 version)
```
