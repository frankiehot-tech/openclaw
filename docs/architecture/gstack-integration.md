# gstack决策框架集成指南

## 概述
gstack是一个工程化决策框架，为OpenClaw项目提供智能决策支持。本框架基于MAREF三才六层模型构建，通过系统性原则和决策树提升项目的工程化水平。

## 核心原则
gstack包含6个核心决策原则：

### 1. 需求评估原则
- **CEO强制性问题**: 在任何需求讨论开始时，自动应用6个CEO强制性问题
- **价值驱动决策**: 明确业务价值和技术价值的平衡点
- **风险前置识别**: 在早期识别技术和业务风险

### 2. 技术设计原则  
- **工程审查清单**: 技术设计阶段自动检查工程审查清单
- **架构模式验证**: 确保设计符合最佳架构模式
- **技术债务评估**: 评估设计决策对技术债务的影响

### 3. UI/UX设计原则
- **10分维度评估**: 用户界面设计自动进行10个维度的可用性评估
- **用户体验流**: 优化用户旅程和交互流程
- **可访问性标准**: 确保界面符合可访问性要求

### 4. 发布管理原则
- **质量门禁检查**: 发布前自动提醒和执行质量门禁检查
- **风险控制矩阵**: 应用风险控制矩阵验证发布准备
- **回滚能力验证**: 确保所有变更具备快速回滚能力

## 集成架构

### 智能触发器
gstack通过智能触发器在特定场景自动应用：
- **需求讨论** → 自动应用CEO强制性问题
- **技术设计** → 自动检查工程审查清单  
- **UI/UX设计** → 自动进行10分维度评估
- **发布准备** → 自动提醒质量门禁

### 知识库位置
- **核心原则**: `~/.claude/knowledge/gstack-core/principles/`
- **决策树**: `~/.claude/knowledge/gstack-core/decision-trees/`
- **智能触发器**: `~/.claude/skills/gstack-intelligence.md`

## 使用指南

### 自动触发模式
gstack框架在以下对话场景中自动触发：

1. **需求讨论场景**
   ```
   用户: "我们需要添加一个新的功能..."
   gstack: [自动应用6个CEO强制性问题]
   ```

2. **技术设计场景**
   ```
   用户: "应该如何设计这个系统架构..."
   gstack: [自动检查工程审查清单]
   ```

3. **发布准备场景**
   ```
   用户: "准备发布新版本..."
   gstack: [自动提醒质量门禁检查]
   ```

### 手动查阅
如需手动查阅gstack原则，可访问以下位置：
- 核心原则文件: `~/.claude/knowledge/gstack-core/principles/`
- 决策树文件: `~/.claude/knowledge/gstack-core/decision-trees/`

## 实际应用案例

### 案例1: 文档迁移项目
在文档迁移计划制定过程中，gstack自动应用了：
- **需求评估**: 6个CEO强制性问题确保迁移目标清晰
- **技术设计**: 工程审查清单验证了分类架构合理性
- **发布管理**: 质量门禁检查确认迁移后的文档系统完整性

### 案例2: 队列监控系统开发
在队列监控系统设计中，gstack提供了：
- **风险识别**: 提前识别了状态同步和认证集成风险
- **技术验证**: 验证了监控仪表板的技术架构
- **发布准备**: 质量门禁确保监控系统稳定可用

## 故障排除

### 常见问题

#### Q1: gstack没有自动触发
**可能原因**: 知识库文件路径不正确
**解决方案**: 检查`~/.claude/knowledge/gstack-core/`目录是否存在

#### Q2: CEO问题没有在需求讨论中应用
**可能原因**: 智能触发器配置问题
**解决方案**: 验证`~/.claude/skills/gstack-intelligence.md`配置

#### Q3: 质量门禁检查遗漏
**可能原因**: 发布场景识别不准确
**解决方案**: 检查gstack触发器配置，确保"发布准备"关键词正确
