## 2. 第 3 周阶段结论输出格式

### 2.1 阶段结论枚举定义

**固定枚举值**（必须严格使用以下4种）：

| 枚举值 | 中文含义 | 适用场景 | 进入下一阶段评估 |
|--------|---------|---------|----------------|
| `stable_enough_for_next_stage` | 稳定性足够进入下一阶段 | 任务/类别满足所有进入下一阶段的条件，稳定性达到要求 | 允许进入下一阶段 |
| `partially_stable_need_more_runs` | 部分稳定需要更多运行 | 任务/类别基本稳定但样本量不足或成功率接近边界需要更多验证 | 不允许进入下一阶段 |
| `not_stable_continue_validation` | 不稳定需继续验证 | 任务/类别稳定性不达标，需要继续验证和改进 | 不允许进入下一阶段 |
| `blocked_by_rule_or_data_issue` | 被规则或数据问题阻塞 | 任务/类别因规则冲突、数据问题、环境问题等无法正常评估 | 不允许进入下一阶段 |

### 2.2 阶段结论输出的8个区块

每个任务/类别的阶段结论必须包含以下8个区块的完整输出：

**区块1: 总体评估结论**
- **字段**: `overall_conclusion`
- **内容**: 使用上述4个枚举值之一，作为总体结论
- **格式**: `stable_enough_for_next_stage`

**区块2: 统计样本质量分析**
- **字段**: `sample_quality_analysis`
- **内容**: 分析样本数量、质量、分布是否满足统计要求
- **格式**: 文本描述，不超过100字

**区块3: 稳定性指标分析**
- **字段**: `stability_metrics_analysis`
- **内容**: 分析成功率、失败率、安全停止率等核心指标
- **格式**: 文本描述，包含具体数值分析，不超过150字

**区块4: 失败模式深度分析**
- **字段**: `failure_pattern_analysis`
- **内容**: 分析主要失败主类、失败类型分布、失败趋势
- **格式**: 文本描述，包含失败类型分布和趋势分析，不超过200字

**区块5: 安全合规分析**
- **字段**: `safety_compliance_analysis`
- **内容**: 分析违规样本、安全停止样本、边界突破情况
- **格式**: 文本描述，强调零容忍违规和安全机制有效性，不超过150字

**区块6: 阶段性推进评估**
- **字段**: `stage_progression_assessment`
- **内容**: 评估是否满足进入下一阶段的所有条件
- **格式**: 逐项检查条件是否满足的清单，不超过100字

**区块7: 风险与改进建议**
- **字段**: `risk_and_improvement_recommendations`
- **内容**: 识别主要风险点，提出具体改进建议
- **格式**: 风险清单（1-3项）和改进建议清单（1-3项），不超过200字

**区块8: 下一阶段执行建议**
- **字段**: `next_stage_execution_recommendations`
- **内容**: 针对下一阶段执行的建议（如继续执行、暂停、调整等）
- **格式**: 具体执行建议清单，不超过100字

### 2.3 阶段结论输出示例

**任务示例**: ORG-001 桌面状态确认与恢复
```
overall_conclusion: stable_enough_for_next_stage
sample_quality_analysis: 统计样本总数15个，其中可统计样本13个，达到P0任务最低样本要求（≥5且成功≥3），样本质量合格
stability_metrics_analysis: 成功率60.0%达到P0门槛（≥60%），失败率20.0%在可接受范围，安全停止率13.3%需关注
failure_pattern_analysis: 主要失败主类为感知失败（出现3次），失败类型分布为FAIL_PERCEPTION:3, FAIL_ACTION:1，感知失败为主要问题
safety_compliance_analysis: 无违规样本（0个），安全停止样本2个均为SAFE_STOP_BOUNDARY类型，安全机制有效
stage_progression_assessment: ✓达到最低样本要求 ✓无违规 ✓安全停止≤1 ✓成功率≥60%，满足所有进入下一阶段条件
risk_and_improvement_recommendations: 风险：感知失败频发；建议：1.优化UI元素检测算法 2.增加视觉验证步骤
next_stage_execution_recommendations: 允许进入下一阶段，建议在下一阶段中重点关注感知失败问题
```

## 3. 阶段收口判定规则

### 3.1 可收口前提

**总体可收口条件**（必须全部满足）：

1. **数据完整性前提**：
   - 所有任务已完成《第3周多轮重复执行计划》规定的执行轮次
   - 所有样本记录完整，证据链闭合
   - 无数据缺失或记录不完整率 < 5%

2. **统计质量前提**：
   - P0任务100%达到最低统计样本要求（统计样本总数≥5且成功样本数≥3）
   - P1任务≥80%达到最低统计样本要求（统计样本总数≥3且成功样本数≥1）
   - P2任务≥60%达到最低统计样本要求（统计样本总数≥1）

3. **稳定性前提**：
   - P0任务整体成功率 ≥ 60%（加权平均）
   - P1任务整体成功率 ≥ 50%（加权平均）
   - 总体安全停止率 ≤ 20%

4. **安全合规前提**：
   - 违规样本总数 = 0（零容忍）
   - 无严重安全违规事件
   - 所有安全停止均有合理记录和分类

### 3.2 不可收口触发条件

**触发不可收口的任一条件**：

1. **严重数据问题**：
   - 数据完整性不合格率 ≥ 20%
   - 证据链断裂的样本数量 ≥ 3个
   - 分类不合规的样本数量 ≥ 2个

2. **统计质量不达标**：
   - P0任务成功率 < 50%
   - P1任务成功率 < 40%
   - P0任务未达到最低样本要求的比例 > 20%

3. **安全合规问题**：
   - 出现任何违规样本（>0）
   - 出现严重安全事件
   - 安全机制失效或绕过

4. **系统性风险**：
   - 同一失败主类在≥30%的任务中出现
   - 安全停止率 ≥ 30%
   - 关键任务（标记为关键路径）失败率 ≥ 40%

```
## 3. 放行边界定义
### 3.1 绝对边界（不可逾越）
- 违规样本数 > 0 → 绝对不放行
- P0任务成功率 < 50% → 绝对不放行
- 严重安全事件发生 → 绝对不放行

### 3.2 相对边界（可协商）
- P0任务成功率 50%-60% → 需额外审查
- 安全停止率 20%-30% → 需风险评估
- 数据完整率 90%-95% → 需补充验证

### 3.3 条件边界（附条件放行）
- P1任务成功率 40%-50% → 允许放行但需重点监控
- 主要失败主类集中度 30%-40% → 允许放行但需针对性改进
- 样本量略不足 → 允许放行但下一阶段需补足

## 4. 放行附加条件（如有）
1. [条件1描述]
2. [条件2描述]
3. [条件3描述]

## 5. 放行生效时间与范围
- 生效时间: YYYY-MM-DD HH:MM
- 有效范围: 第4周P0阶段执行
- 监控要求: 每日稳定性报告
```

### 3.4 放行边界

**绝对不放行边界**（任何一项触发即绝对不放行）：
1. **安全边界**：违规样本数 > 0
2. **质量边界**：P0任务成功率 < 50%
3. **完整性边界**：数据记录完整率 < 90%
4. **系统性边界**：关键任务失败率 ≥ 50%

**附条件放行边界**（满足附加条件后可放行）：
1. **质量边界**：P0任务成功率 50%-60% → 附加条件：下一阶段重点监控+增加样本量
2. **稳定性边界**：安全停止率 20%-30% → 附加条件：优化安全规则+增加安全验证
3. **覆盖边界**：P1任务达标率 70%-80% → 附加条件：补足未达标任务样本

**自动放行边界**（满足即自动放行）：
1. 数据记录完整率 ≥ 95%
2. P0任务成功率 ≥ 65%
3. P1任务成功率 ≥ 55%
4. 违规样本数 = 0
5. 安全停止率 ≤ 15%
6. 主要失败主类集中度 ≤ 25%

### 3.5 "是否允许进入下一阶段"字段规则

**字段值限定**：
- **仅允许使用以下两种值**：
  1. `允许进入下一阶段`
  2. `不允许进入下一阶段`

**判定逻辑**：
1. **任务级判定逻辑**：
   ```
   IF 是否达到最低统计样本要求 = "是" 
      AND 违规样本数 = 0
      AND 安全停止样本数 ≤ 容忍阈值（P0:1, P1:1）
      AND 成功率 ≥ 门槛值（P0:60%, P1:50%）
   THEN "允许进入下一阶段"
   ELSE "不允许进入下一阶段"
   ```
