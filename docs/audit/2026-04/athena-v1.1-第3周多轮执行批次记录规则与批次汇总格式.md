## 批次记录规则

### 批次定义与批次编号规则

#### 批次定义
批次是按《Athena_v1.1_第3周多轮重复执行计划与样本统计口径.md》划分的执行单元，包含同一批次内的多个任务执行样本。一个批次代表一次连续执行周期，覆盖相同优先级层级的多个任务。

#### 批次编号规则
- **格式**: `B01`, `B02`, `B03` ...（两位数字，从01开始顺序递增）
- **分配原则**: 按执行时间顺序分配，先执行的批次编号小
- **示例**: `B01`, `B02`, `B03`

#### 批次状态枚举
| 状态代码 | 状态名称 | 状态含义 | 允许的状态流转 |
|---------|---------|---------|---------------|
| `planned` | 已计划 | 批次计划已创建，尚未开始执行 | → `in_progress` |
| `in_progress` | 执行中 | 批次内任务正在执行 | → `ready_for_batch_review`, `batch_unqualified` |
| `ready_for_batch_review` | 准备批次审核 | 批次执行完成，等待批次级审核 | → `batch_completed`, `batch_unqualified` |
| `batch_unqualified` | 批次不合格 | 批次审核未通过，不允许并入任务级统计 | → `in_progress`（修正后） |
| `batch_completed` | 批次已完成 | 批次审核通过，允许并入任务级统计 | 终端状态 |

### 批次记录创建与更新规则

#### 批次记录创建时机
1. **批次计划阶段**: 根据《第3周多轮重复执行计划》创建批次记录，状态为`planned`
2. **批次执行开始时**: 更新状态为`in_progress`，记录计划执行日期
3. **批次执行过程中**: 实时更新样本编号列表、样本数量等字段

#### 批次记录更新频率
- **实时更新**: 每完成一个样本，立即更新批次内样本编号列表
- **每日更新**: 每日结束时更新批次状态和关键统计字段
- **批次完成时**: 批次内所有任务执行完毕后，立即更新状态为`ready_for_batch_review`

#### 批次状态流转规则
1. `planned` → `in_progress`: 批次开始执行时
2. `in_progress` → `ready_for_batch_review`: 批次内所有任务执行完毕
3. `ready_for_batch_review` → `batch_completed`: 批次审核通过
4. `ready_for_batch_review` → `batch_unqualified`: 批次审核未通过
5. `in_progress` → `batch_unqualified`: 批次执行中出现严重问题
6. `batch_unqualified` → `in_progress`: 修正后重新执行

## 批次汇总格式

### 批次汇总字段定义

| 字段名称 | 字段用途 | 字段边界 | 数据类型 | 必填性 | 示例值 |
|---------|---------|---------|---------|--------|--------|
| 批次编号 | 关联批次记录 | 与批次记录中的批次编号一致 | 字符串 | 必填 | B01 |
| 覆盖任务数 | 批次覆盖的任务数量 | 批次内去重后的任务编号数量 | 整数 ≥0 | 必填 | 3 |
| 覆盖样本数 | 批次执行的总样本数量 | 与批次记录中的批次样本总数一致 | 整数 ≥0 | 必填 | 15 |
| 可统计样本数 | 批次内符合统计标准的样本数量 | 与批次记录中的批次可统计样本数一致 | 整数 ≥0 | 必填 | 13 |
| 成功样本数 | 批次内成功样本数量 | 按《第3周稳定性统计字段与计算规则》成功样本定义 | 整数 ≥0 | 必填 | 9 |
| 普通失败样本数 | 批次内普通失败样本数量 | 按《第3周稳定性统计字段与计算规则》普通失败样本定义 | 整数 ≥0 | 必填 | 3 |
| 安全停止样本数 | 批次内安全停止样本数量 | 按《第3周稳定性统计字段与计算规则》安全停止样本定义 | 整数 ≥0 | 必填 | 1 |
| 违规样本数 | 批次内违规样本数量 | 与批次记录中的批次违规样本数一致 | 整数 ≥0 | 必填 | 0 |
| 不计入统计样本数 | 批次内不计入统计的样本数量 | 与批次记录中的批次不计入统计样本数一致 | 整数 ≥0 | 必填 | 2 |
| 批次成功率 | 批次内成功样本占可统计样本数的比例 | 计算公式：成功样本数 / 可统计样本数 × 100% | 百分比 (0-100) | 必填 | 69.2% |
| 批次失败率 | 批次内普通失败和安全停止样本占可统计样本数的比例 | 计算公式：(普通失败样本数 + 安全停止样本数) / 可统计样本数 × 100% | 百分比 (0-100) | 必填 | 30.8% |
| 批次安全停止率 | 批次内安全停止样本占可统计样本数的比例 | 计算公式：安全停止样本数 / 可统计样本数 × 100% | 百分比 (0-100) | 必填 | 7.7% |
| 批次违规率 | 批次内违规样本占可统计样本数的比例 | 计算公式：违规样本数 / (可统计样本数 + 违规样本数 + 不计入统计样本数) × 100% | 百分比 (0-100) | 必填 | 0.0% |
| 批次主要失败主类 | 批次内普通失败样本中出现最多的失败主类 | 从《Athena失败taxonomy冻结版.md》5大失败主类中选择 | 字符串 | 条件必填（当普通失败样本数>0时） | 感知失败 |
| 批次是否允许并入任务级统计 | 判断批次数据是否允许并入任务级统计 | 判定条件：批次状态=batch_completed AND 批次违规样本数=0 AND 批次可统计样本数≥批次目标样本数的80% | 布尔值 | 必填 | 是 |
| 批次是否需要进入修正清单 | 判断批次是否需要进入修正清单 | 判定条件：批次状态=batch_unqualified OR 批次违规样本数>0 | 布尔值 | 必填 | 否 |
| 备注 | 补充说明和异常记录 | 记录批次汇总的特殊情况、数据异常等，不超过200字 | 字符串 | 可选 | 批次成功率未达到70%目标，需关注感知失败问题 |

### 批次汇总计算规则

#### 计算口径一致性原则
批次级计算口径必须与《Athena_v1.1_第3周稳定性统计字段与计算规则》中定义的任务级统计口径完全一致：

1. **分子分母定义一致**：
   - 成功率分子：成功样本数
   - 成功率分母：可统计样本数（不是批次样本总数）
   - 失败率分子：普通失败样本数
   - 失败率分母：可统计样本数
   - 安全停止率分子：安全停止样本数
   - 安全停止率分母：可统计样本数
   - 违规率分子：违规样本数
   - 违规率分母：可统计样本数

2. **样本分类标准一致**：
   - 成功样本：按《第3周多轮重复执行计划》2.2节"成功样本"定义
   - 普通失败样本：按《第3周多轮重复执行计划》2.2节"普通失败样本"定义
   - 安全停止样本：按《第3周多轮重复执行计划》2.2节"安全停止样本"定义
   - 违规样本：按《第3周多轮重复执行计划》2.2节"违规样本"定义
   - 不计入统计样本：按《第3周多轮重复执行计划》2.1节"不计入统计的样本"定义

#### 批次主要失败主类判定规则
- 仅当批次普通失败样本数 > 0 时需填写此字段
- 从批次内所有普通失败样本的失败主类中，选择出现次数最多的主类
- 如果多个主类出现次数相同，按以下优先级选择：感知失败 > 动作失败 > 状态不匹配 > 不安全继续 > 超时

#### 批次合格性判定规则
**批次允许并入任务级统计的条件**（必须全部满足）：
1. 批次状态 = `batch_completed`
2. 批次违规样本数 = 0
3. 批次可统计样本数 ≥ 批次目标样本数的80%
4. 根据批次优先级层级，满足相应的成功率要求：
   - P0批次：批次成功率 ≥ 60%
   - P1批次：批次成功率 ≥ 50%
   - P2批次：无强制成功率要求

**批次需要进入修正清单的条件**（满足任一条件）：
1. 批次状态 = `batch_unqualified`
2. 批次违规样本数 > 0
3. 批次可统计样本数 < 批次目标样本数的50%
4. 根据批次优先级层级，成功率严重不达标：
   - P0批次：批次成功率 < 40%
   - P1批次：批次成功率 < 30%

### 批次汇总输出格式

#### 批次汇总表（Excel/CSV格式）
**文件名**: `Athena_v1.1_Week3_Batch_Summary_YYYYMMDD.csv`
**字段顺序**: 必须按批次汇总字段定义顺序排列
**编码**: UTF-8 with BOM
**分隔符**: 逗号（CSV）
**更新频率**: 每个批次状态变为`ready_for_batch_review`时输出或更新

2. 按优先级分层统计（P0/P1/P2批次分别统计）
3. 批次合格率分析
4. 批次失败模式分析
5. 批次违规情况分析
6. 批次修正清单（如有）
7. 批次数据并入任务级统计计划
8. 建议与改进措施

## 批次与任务级统计衔接规则

### 数据更新顺序与依赖关系

#### 严格的数据更新顺序
所有数据必须按以下顺序更新，确保数据一致性和完整性：

1. **样本记录** → 2. **失败/违规记录** → 3. **批次记录** → 4. **批次汇总** → 5. **任务级统计**

#### 各阶段具体规则
**阶段1: 样本记录**
- **时机**: 每个样本执行完成后立即记录
- **输出**: 《Athena统一测试记录模板.md》填写完整的记录文件
- **验证**: 样本必须通过双重验证（执行人员+独立验证人员）
- **分类**: 按《Athena失败taxonomy冻结版.md》准确分类

**阶段2: 失败/违规记录**
- **时机**: 样本记录完成后，立即进行失败/违规判定
- **输出**: 更新样本记录中的失败分类和违规标记
- **验证**: 失败分类必须与证据链一致，违规判定必须严格按规则

**阶段3: 批次记录**
- **时机**: 批次内所有样本记录完成后，立即更新批次记录
- **输出**: 更新批次记录中的所有字段，状态更新为`ready_for_batch_review`
- **验证**: 批次内样本编号列表必须完整，样本数量必须准确

**阶段4: 批次汇总**
- **时机**: 批次记录完成后，立即计算批次汇总
- **输出**: 生成批次汇总数据
- **验证**: 批次汇总计算必须通过独立公式验证

**阶段5: 任务级统计**
- **时机**: 批次状态为`batch_completed`且通过批次审核后
- **输出**: 将批次数据并入任务级统计表
- **验证**: 批次必须满足"允许并入任务级统计"条件

### 批次不合格处理规则

#### 批次不合格的判定
批次在以下情况下判定为不合格，**不得直接并入任务级统计**：

1. **数据完整性不合格**：
   - 批次内样本记录不完整率 > 20%
   - 批次内证据链断裂的样本数量 > 2个
   - 批次内分类不合规的样本数量 > 1个

2. **统计质量不合格**：
   - 批次可统计样本数 < 批次目标样本数的50%
   - 根据批次优先级层级，成功率严重不达标：
     - P0批次：批次成功率 < 40%
     - P1批次：批次成功率 < 30%

3. **安全合规不合格**：
   - 批次违规样本数 > 0
   - 批次内出现严重安全违规行为

#### 不合格批次处理流程
1. **立即停止**: 发现批次不合格，立即停止该批次数据向任务级统计的流转
2. **状态标记**: 批次状态标记为`batch_unqualified`
3. **根本原因分析**: 进行不合格批次根本原因分析
4. **修正方案制定**: 制定具体修正方案，记录在修正清单中
5. **重新执行或数据修正**: 根据修正方案重新执行或修正数据
6. **重新验证**: 修正后重新验证批次合格性
7. **重新审核**: 验证通过后重新进行批次审核

#### 修正清单管理
**修正清单字段**:
- 批次编号
- 不合格原因分类（数据完整性、统计质量、安全合规）
- 具体问题描述
- 影响范围（影响的样本编号、任务编号）
- 修正方案（重新执行、数据修正、规则调整）
- 责任人
- 计划完成时间
- 实际完成时间
- 修正结果验证

**修正清单输出**: `Athena_v1.1_Week3_Batch_Correction_List_YYYYMMDD.md`

### 批次数据并入任务级统计规则

#### 并入条件（必须全部满足）
1. **批次状态条件**: 批次状态 = `batch_completed`
2. **批次合格性条件**: 批次是否允许并入任务级统计 = `是`
3. **数据完整性条件**: 批次内所有样本记录完整，证据链闭合
4. **时间顺序条件**: 批次执行时间在任务级统计时间范围内

#### 并入操作规则
1. **样本级并入**: 将批次内每个可统计样本按任务编号归类
2. **任务统计更新**: 更新对应任务的统计字段：
   - 统计样本总数 += 批次内该任务的可统计样本数
   - 成功样本数 += 批次内该任务的成功样本数
   - 普通失败样本数 += 批次内该任务的普通失败样本数
   - 安全停止样本数 += 批次内该任务的安全停止样本数
   - 违规样本数 += 批次内该任务的违规样本数
3. **重新计算指标**: 重新计算任务的成功率、失败率、安全停止率、违规率
4. **更新评估字段**: 重新计算"是否达到最低统计样本要求"和"是否允许进入下一阶段评估"

#### 并入冲突处理
1. **样本重复冲突**: 如果样本已存在于任务级统计中，以最早记录的样本为准，后续重复样本标记为"不计入统计"
2. **数据不一致冲突**: 如果批次数据与现有任务级统计数据不一致，优先采用证据链更完整的版本
3. **时间范围冲突**: 如果批次执行时间超出统计时间范围，需要特殊审批后方可并入

#### 并入审计记录
每次批次数据并入任务级统计必须记录审计日志：
- 并入批次编号
- 并入时间
- 并入操作人员
- 影响的任务编号列表
- 并入前后数据对比
- 特殊处理说明（如有）

### 衔接质量保障措施

#### 数据一致性检查
批次数据并入任务级统计前必须进行一致性检查：
1. **样本编号唯一性检查**: 确保样本编号在任务级统计中唯一
2. **任务编号有效性检查**: 确保任务编号在任务定义中存在
3. **分类合规性检查**: 确保所有样本分类符合taxonomy标准
4. **计算公式验证**: 验证批次和任务级统计计算公式一致性

#### 自动化验证规则
建议实现以下自动化验证规则：
1. **批次状态流转验证**: 验证批次状态流转符合规则
2. **数据更新顺序验证**: 验证数据更新符合规定的顺序
3. **合格性条件验证**: 自动验证批次是否满足并入条件
4. **冲突检测预警**: 自动检测数据冲突并预警

#### 人工审核要点
每个批次数据并入任务级统计前必须进行人工审核：
1. **批次合格性审核**: 审核批次是否真正满足并入条件
2. **数据完整性审核**: 审核批次内所有样本记录是否完整
3. **证据链审核**: 审核关键样本的证据链是否闭合
4. **分类准确性审核**: 审核样本分类是否准确合理
