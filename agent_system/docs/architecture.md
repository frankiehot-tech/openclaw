# Agent 控制链路架构文档

## 概述

本系统实现了一个多层次的 Agent 控制链路，使 Athena Agent 能够通过 AutoGLM 智能决策来控制 Samsung Galaxy Z Flip3 设备。

## 系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        Athena Agent (本地)                       │
│                      任务入口 / 任务分发                          │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                      Athena Adapter Layer                        │
│                   任务路由 / 接口转换                             │
│                   - athena_interface.py                         │
│                   - task_router.py                               │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                      AutoGLM Bridge Layer                        │
│                   UI理解 + 动作决策                               │
│                   - agent_loop.py (核心循环)                     │
│                   - model_client.py (模型调用)                   │
│                   - action_executor.py (动作执行)                │
│                   - memory.py (历史记录)                         │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                      Device Control Layer                        │
│                   ADB 封装 / 设备操作                             │
│                   - adb_client.py                                │
│                   - screen_capture.py                            │
│                   - device_manager.py                            │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                    Samsung Galaxy Z Flip3                        │
│                      被控设备                                    │
└─────────────────────────────────────────────────────────────────┘
```

## 数据流

### 任务流转

1. **Athena → Athena Adapter**
   - 接收自然语言任务
   - 格式：`{"task": "...", "context": "...", "device": "zflip3"}`

2. **Athena Adapter → AutoGLM Bridge**
   - 转换任务格式
   - 添加设备上下文

3. **AutoGLM Bridge → Device Control**
   - 截图获取当前UI状态
   - 发送截图+任务给模型推理
   - 接收动作决策
   - 执行动作

4. **Device Control → 设备**
   - 通过 ADB 命令执行操作

## 控制循环

```
┌──────────────────────────────────────────┐
│           开始任务循环                     │
└─────────────────┬────────────────────────┘
                  │
                  ▼

                  ▼

                  │
                  ▼

│         3. execute_action()              │

                  │

                  │
                  ▼
         ┌────────┴────────┐
         │ 任务完成?        │
         └────────┬────────┘
            是    │    否
             ▼    │    │
    ┌─────────────┐ │    │
    │   结束循环   │ │    │
    └─────────────┘ │    │
                    ▼    │
           ┌──────────────┐
           │ 返回步骤1    │
           └──────────────┘
```

## 模块职责

### 1. Athena Adapter (athena_adapter/)

| 文件 | 职责 |
|------|------|
| `athena_interface.py` | 对外统一入口，接收任务 |
| `task_router.py` | 任务路由与格式转换 |

### 2. AutoGLM Bridge (autoglm_bridge/)

| 文件 | 职责 |
|------|------|
| `agent_loop.py` | 核心控制循环 |
| `model_client.py` | 模型API封装 |
| `action_executor.py` | 动作执行器 |
| `memory.py` | 历史步骤记录 |

### 3. Device Control (device_control/)

| 文件 | 职责 |
|------|------|
| `adb_client.py` | ADB命令封装 |
| `screen_capture.py` | 屏幕截图 |
| `device_manager.py` | 设备管理 |

### 4. Scripts (scripts/)

| 文件 | 职责 |
|------|------|
| `check_device.sh` | 设备状态检查 |

## 统一接口协议

### 任务输入格式

```json
{
  "task": "打开设置并查看关于手机",
  "context": "用户想要了解设备信息",
  "device": "zflip3"
}
```

### 模型输出格式

```json
{
  "action": "tap",
  "x": 500,
  "y": 1200,
  "text": null,
  "reasoning": "需要点击设置图标"
}
```

支持的 action 类型：
- `tap`: 点击 (x, y)
- `swipe`: 滑动 (x1, y1) → (x2, y2)
- `input`: 输入文本
- `back`: 返回
- `home`: 主页

## 设备支持

当前支持设备：
- Samsung Galaxy Z Flip3 (代号: `zflip3`)

架构设计支持未来多设备扩展，通过 `device_manager.py` 管理。

## 安全机制

1. **动作校验**: 执行前验证坐标是否在屏幕范围内
2. **重试机制**: 失败操作自动重试 (默认3次)
3. **超时控制**: 单次操作超时限制 (默认30秒)
4. **循环检测**: 通过 memory 防止无限循环
