## 2. 测试记录模板

```markdown
## 测试记录
- 任务名称：
- 任务类别：导航 / 查找 / 整理
- 输入目标：
- 起始截图：
- 执行动作序列：
- 结束截图：
- 是否成功：成功 / 失败
- 失败类型：perception_failed / action_failed / state_mismatch / timeout / unsafe_to_continue
- 是否安全停止：是 / 否
- 备注：
```

**字段说明**：
- **任务名称**：测试的具体任务名称
- **任务类别**：导航、查找、整理三类之一
- **输入目标**：任务的具体目标描述
- **起始截图**：任务开始时的屏幕截图文件路径
- **执行动作序列**：按顺序记录每个执行的动作
- **结束截图**：任务结束时的屏幕截图文件路径
- **是否成功**：任务最终是否成功完成
- **失败类型**：
  - `perception_failed`：感知失败，无法正确识别屏幕内容
  - `action_failed`：动作执行失败，点击/滑动等操作无效
  - `state_mismatch`：状态不匹配，预期状态与实际状态不符
  - `timeout`：超时，在规定时间内未完成任务
  - `unsafe_to_continue`：不安全继续，检测到可能的风险
- **是否安全停止**：失败时是否安全停止，没有造成额外损害
- **备注**：其他需要记录的信息
